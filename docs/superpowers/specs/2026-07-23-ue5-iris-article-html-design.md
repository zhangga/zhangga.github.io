# UE5 Iris 知乎文章静态 HTML 设计

## 目标

将知乎文章《UE5-Iris 网络复制系统技术分析指南 - 大纲》转换为可独立访问的静态 HTML 页面，放入本站 `source/html-articles` 目录。页面完整保留原文内容与层级，并清晰注明作者和文章来源。

## 输出

- 主页面：`source/html-articles/ue5-iris-guide-outline/index.html`
- 本地资源：`source/html-articles/ue5-iris-guide-outline/assets/`
- 不新增 Hexo `_posts` 入口，不修改站点配置。

## 内容处理

- 保留标题、引言、章节标题、列表、表格、分隔线、链接和原文插图。
- 仅移除知乎导航、推荐阅读、评论、互动按钮等平台界面内容。
- 保持原文的章节顺序和文字，不自行补写技术结论。
- 将原文正文中的站内搜索链接保留为可访问的外部链接。
- 将正文图片保存到本地 `assets/` 并改写引用，页面不依赖知乎图片地址。

## 来源与署名

页首和页尾均展示：

- 原文标题：《UE5-Iris 网络复制系统技术分析指南 - 大纲》
- 作者：Smartuil
- 来源：知乎专栏
- 原文链接：`https://zhuanlan.zhihu.com/p/1996685633524089868`
- 原文更新时间：以知乎页面显示的信息为准

原文链接使用可点击链接，并在新标签页打开。

## 页面设计

采用独立的编辑式技术文档风格，不复刻知乎界面：

- 桌面端为左侧固定目录、右侧正文布局。
- 移动端将目录折叠为顶部可展开区域。
- 正文使用适合中文长文的字体栈、舒适行高和受控内容宽度。
- 章节层级、代码、表格、引用和链接具有清晰区分。
- 页面不依赖第三方 JavaScript 或远程字体；目录高亮和展开行为使用少量原生 JavaScript。
- 尊重 `prefers-reduced-motion`，并提供键盘可用的目录导航和明显焦点样式。

## 验收标准

- HTML 使用 UTF-8，在主流桌面和移动浏览器中可直接打开。
- 页面完整包含知乎文章正文，不混入评论、推荐或平台导航。
- 目录链接能够跳转到对应章节，标题 ID 唯一。
- 来源和作者在页首、页尾均清晰可见。
- 本地图片路径有效，页面不存在意外的知乎界面资源依赖。
- `npm run build` 成功，生成后的文件位于 `public/html-articles/ue5-iris-guide-outline/`。
- 通过浏览器检查桌面端和窄屏布局、目录导航、链接及中文显示。
